# Security Policy

We take the security of BeamFlow seriously. Thank you for helping us keep this project safe and secure.

---

## Supported Versions

Currently, security updates and patches are active for the following versions of BeamFlow:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability in this project, please report it by emailing us at [security@beamflow.org](mailto:security@beamflow.org). 

In your email, please include:
- A description of the vulnerability and its potential impact.
- Detailed step-by-step instructions (or a proof-of-concept script/workflow) to reproduce the vulnerability.
- Any potential configurations or environments where the vulnerability might occur.

### Our Response Process

1. **Receipt**: We will acknowledge receipt of your vulnerability report within 48 hours.
2. **Evaluation**: We will evaluate the report and confirm the vulnerability.
3. **Resolution**: If confirmed, we will work on a fix. We aim to keep you informed of our progress.
4. **Disclosure**: Once the fix is released, we will coordinate with you to decide on a public disclosure timeframe. We request that you do not disclose the vulnerability publicly until a fix is available and published.

*Note: Please update the contact email above if you prefer security reports to be sent to a different address.*
